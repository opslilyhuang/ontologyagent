# 自动化本体发现模块 - 产品集成方案

## 📌 方案概述

### 背景
现有产品已具备完整的本体管理和智能问答能力，但**本体构建环节依赖人工编排**，存在以下痛点：
- ⏰ **效率低**：每个数据源需要人工定义类、属性、关系，耗时长
- 🎯 **门槛高**：需要用户具备本体建模知识，学习成本高
- 🔄 **难扩展**：面对批量数据源，人工编排难以快速响应

### 解决方案
新增**"自动化本体发现"**模块，提供**从数据到知识的智能化通道**：

```
传统流程（人工）:
数据上传 → 手动定义本体 (类/属性/关系) → 数据导入 → 本体管理 → 智能问答
         ↑___________________________↑
              耗时、易错、门槛高

新增流程（AI自动化）:
数据接入 → AI自动分析 → 快速审核确认 → 智能导入 → 本体管理 → 智能问答
         ↑_________________↑
          快速、准确、零门槛
```

## 🎨 关键功能设计

### 功能 1：智能本体发现（AI核心能力）

#### 功能描述
用户上传数据文件后，AI自动分析数据结构，智能生成本体定义。

#### 用户看到的效果

**输入**: 一个 Excel 文件 `customers.xlsx`
```
| 客户ID | 客户名称 | 联系电话      | 电子邮箱            | 注册日期   |
|--------|---------|-------------|-------------------|-----------|
| 1001   | 张三     | 13800138000 | zhang@example.com | 2025-01-15|
| 1002   | 李四     | 13900139000 | li@example.com    | 2025-02-20|
```

**AI输出** (自动生成):
```
本体: 客户本体 (Customer Ontology)

类 (Class): Customer
  属性 (Properties):
    • 客户ID
      - 英文名: customer_id
      - 数据类型: int32
      - 主键: ✓
      - 展示名称: 客户编号
      - 描述: 客户的唯一标识符

    • 客户名称
      - 英文名: customer_name
      - 数据类型: varchar
      - 展示名称: 姓名

    • 联系电话
      - 英文名: phone
      - 数据类型: varchar (自动识别为电话格式)
      - 展示名称: 手机号码

    • 电子邮箱
      - 英文名: email
      - 数据类型: email (自动识别为邮箱格式)
      - 展示名称: 邮箱地址

    • 注册日期
      - 英文名: register_date
      - 数据类型: date (自动识别为日期格式)
      - 展示名称: 注册时间
```

#### AI智能点
- ✅ 自动识别主键（客户ID）
- ✅ 自动推断数据类型（int, varchar, email, date）
- ✅ 自动生成中文标签（客户ID → "客户编号"）
- ✅ 自动生成展示名称（便于前端展示）
- ✅ 自动检测特殊格式（邮箱、电话、URL、IP）

### 功能 2：人工审核与快速编辑

#### 设计理念
AI提供**90%的准确度**，人工审核补充**最后10%的业务理解**。

#### 可编辑项

**类 (Class) 编辑**:
- ✏️ 修改类名和描述
- 🔄 使用自然语言重新分析（如："请将客户和订单建立关联"）

**属性 (Property) 编辑**:

| 字段 | 默认值 | 用户可编辑 | 编辑方式 |
|------|-------|-----------|---------|
| 英文名 | customer_id | ✅ | 文本框输入 |
| 中文标签 | 客户ID | ✅ | 自动生成，可修改 |
| 展示名称 | 客户编号 | ✅ | **新增：文本框输入** |
| 数据类型 | int32 | ✅ | 下拉选择 |
| 主键 | ✓ | ✅ | **新增：复选框** |
| 来源字段 | 客户ID | ❌ | 只读 |

**关键改进**:
1. **主键可手工调整**
   - 场景：AI误判主键，用户可通过复选框快速调整
   - 位置：属性表格中，独立一列

2. **展示名称独立编辑**
   - 场景：中文标签用于内部，展示名称面向最终用户
   - 位置：主键列右侧，可随时修改
   - 示例：
     - 中文标签: "客户ID"
     - 展示名称: "客户编号" / "会员号" / "用户ID"

**关系 (Relation) 编辑**:
- ➕ 添加新关系
- ✏️ 修改关系类型（1:1, 1:n, n:1, n:m）
- 🔗 调整关联字段
- ❌ 删除不合理的关系

### 场景 1: 快速构建客户知识图谱

**用户**: 企业CRM管理员
**需求**: 将现有的客户Excel表导入系统，快速构建知识图谱

**操作步骤**:
1. 上传 `customers.xlsx`（包含1000条客户数据）
2. 点击"开始分析"，等待30秒
3. 审核AI生成的本体，调整主键和展示名称
4. 选择清洗规则（去重、去空、验证邮箱）
5. 点击"确认并导入"
6. 等待20秒，导入完成（成功980条，失败20条）
7. 进入本体管理器，查看客户知识图谱

**总耗时**: 3分钟
**效果**: 客户数据立即可用于智能问答

**用户**: 数据分析师

**效果**: 构建完整的业务知识图谱（客户-订单-产品）
